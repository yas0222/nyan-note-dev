# 猫の健康共有アプリ（MVP設計）

このリポジトリは、**シンプル操作で猫の健康記録を共有できるアプリ**の最小実装（MVP）を設計するためのドキュメントです。

## 1. アプリの目的
- 飼い主が毎日の健康情報を1分以内で記録できる
- 自分の猫の健康推移を見える化できる
- 地域別・全国の傾向を比較できる
- 公開したくない情報は匿名化して共有できる

## 2. 想定ユーザー
- 1〜3匹の猫を飼っている個人ユーザー
- 保護猫団体・地域コミュニティの参加者

## 3. MVPで実現する機能

### 3.1 猫プロフィール登録（必須）
- 写真
- 名前
- 年齢
- 性別
- 地域（都道府県 / 市区町村）
- 公開設定
  - `公開`（名前表示）
  - `匿名公開`（名前を「にゃんこA」などに置換）
  - `非公開`（統計にのみ利用）

### 3.2 毎日記録（必須）
- 1日のエサ総量（g）
- カリカリ比率（%）
- ウェット比率（%）
- おやつ量（g）
- うんち回数
- おしっこ回数
- メモ（任意）

### 3.3 閲覧機能（必須）
- 自分の猫の記録一覧（直近7日 / 30日）
- 他地域の公開データ一覧（匿名を含む）
- 地域統計カード
  - 平均エサ量
  - カリカリ/ウェット平均比率
  - 平均おやつ量
  - うんち・おしっこ平均回数
- 全国統計カード（同項目）

## 4. 画面構成（シンプル導線）
1. **ホーム**: 今日の記録ボタン + 統計サマリー
2. **猫プロフィール**: 登録・編集
3. **記録入力**: 1画面で完結（スライダー中心）
4. **みんなの猫**: 地域フィルタつき一覧
5. **統計**: 地域/全国の比較表示

## 5. UX方針（使い方をシンプルに）
- 記録画面は1画面完結
- 数値入力はステッパー / スライダーで最小タップ
- 前日値を初期値として自動セット
- 「保存して終了」を目立たせる

## 6. プライバシー方針
- 匿名公開時は名前を保存時にマスク表示名へ変換
- 非公開でも統計には個人特定不可の形で集計
- 地域表示は粗い粒度（市区町村まで）をデフォルト

## 7. 技術構成（提案）
- フロント: React Native (Expo)
- バックエンド: Supabase (Auth + PostgreSQL + Storage)
- 画像保存: Supabase Storage
- 統計集計: SQL View + Materialized View（定期更新）

詳細データ設計は `docs/data-model.md` を参照。
